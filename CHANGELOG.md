## [0.1.1] - 2026-07-17

A correctness, safety, and supply-chain patch release: the first post-v0.1.0
deep review (14 module reviewers, adversarially verified) found a batch of
cross-crate seam and lifecycle defects, all fixed here test-first.

### Security

- **ci**: The release pipeline no longer interpolates `github.*` context (the
  tag name) into privileged `run:` bodies — a crafted tag could otherwise inject
  shell into the job that holds the minisign secret. It now re-runs the full CI
  gate (cargo-deny + clippy + tests) on the tagged commit before publishing,
  wipes the signing key on the failure path, pins the Inno Setup version with a
  required checksum, and builds hermetically without a mutable cache (#47)
- **platform,app**: `dujactl` now verifies the pipe server's token-user SID
  before trusting a connection, so a same-user pipe squatter can no longer
  intercept control traffic; on mismatch it falls back to the direct backend (#55)

### Fixed

- **dujactl**: `set all` works over IPC again — it reconnects per request instead
  of reusing a single one-request-per-connection stream (it silently failed
  against a running app before) (#46)
- **app**: Worker lifecycle is safe on detach, open-failure, and quit — a bounded
  shutdown join no longer hangs app exit on a wedged driver call, a detached
  worker can never become a second writer to a panel, a failed controller open
  greys the display and recovers instead of silently losing control, and
  watchdog recovery restores the user's level instead of relearning it
  (ADR-0017) (#48)
- **dimmer**: Overlay windows are destroyed on a creation-failure path (no more
  leaked windows), capture-exclusion failure degrades to a working dim instead of
  none, the gamma crash marker is preserved when a restore fails (the never-brick
  net), and overlay apply/clear can no longer freeze the UI thread on a wedged
  worker (#49)
- **app**: The HDR gamma verdict is re-probed live instead of frozen at launch,
  so turning HDR on after start routes to the overlay instead of an ineffective
  gamma ramp; `begin_quit` preserves the crash marker on a failed restore (#53)
- **core**: Serial-less display identity keys on the 128-byte EDID base block
  only, so a truncated or extension-carrying read of the same panel no longer
  splits its identity and loses its per-monitor config (#50)
- **ui**: A hot-plug during a slider drag no longer retargets the drag to the
  wrong monitor (the row model is order-stable by id); the settings input dropdown
  keeps its selection and stops resetting mid-click; the resize grip no longer
  hijacks scrollbar drags (#52)
- **ddc**: A hardware-reported continuous max of 0 is floored to 1, so a corrupt
  DDC reply can no longer drive a panel to its darkest level regardless of the
  slider (#54)
- **app**: The update check treats a future last-check timestamp as due (a
  backward clock can no longer wedge it off), motion detection defaults to on when
  the OS query fails, logging falls back to the temp root when no home dir
  resolves, and the open flyout re-applies its work-area height cap on a
  resize-while-visible so it can't grow off-screen (#51)
- **platform**: The Windows installer detects a running instance via a fixed-name
  guard mutex (`AppMutex`) and prompts to close it, instead of silently failing an
  in-place upgrade (#55)

### Changed

- **ci**: Release checksums are written LF-only so `sha256sum -c` passes on Linux
  and macOS; a drop-in, secret-gated Azure Trusted Signing step is staged so
  Authenticode signing can be enabled later with no workflow edit (#47)

### Documentation

- **adr**: ADR-0017 (engine shutdown & worker-lifecycle contract), ADR-0018
  (app-owns-continuum / engine-owns-pacing), ADR-0019 (version ladder & release
  trains), ADR-0020 (release integrity & signing readiness); a new
  `docs/release-checklist.md` runbook
